# markov.misc Architecture

## Overview

`markov.misc` is an R package designed for the simulation, analysis, and visualization of longitudinal ordinal outcomes in clinical trials. Its primary focus is on discrete-time Markov transition models, specifically using (partial) proportional odds logistic regression to estimate transition probabilities between health states (e.g., home, hospital, ICU, death).

The package bridges the gap between flexible regression models (like those from `VGAM` and `rms`) and practical clinical trial quantities of interest, such as State Occupancy Probabilities (SOPs), Time in State, and Days Alive and Out of Hospital (DAOH). It provides a robust pipeline for:

1.  **Simulating** realistic patient trajectories under complex assumptions.
2.  **Fitting** transition models with robust covariance estimation.
3.  **Estimating** marginal quantities via G-computation.
4.  **Inferring** uncertainty using fast MVN simulation or bootstrap methods.
5.  **Assessing** operating characteristics (power/Type I error) for trial design.

## Core Components

### 1. Patient Simulation Engines

The package offers three distinct methodologies for generating longitudinal health state data, located primarily in `R/simulate_trajectories.R`.

-   **Markov Simulator** (`sim_trajectories_markov`): Implements a discrete-time first-order Markov model where transition probabilities are governed by a proportional odds model. It supports time-varying covariates, custom linear predictor functions (e.g., `lp_violet`), and interactions between previous state and time.
-   **Latent Brownian Simulator** (`sim_trajectories_brownian`): Models health status as a continuous latent variable evolving via a Gaussian random walk with drift. Observed states are generated by thresholding this latent variable. This is useful for testing model robustness against misspecification (non-Markovian data).
-   **Latent Brownian Gap Simulator** (`sim_trajectories_brownian_gap`): Uses the same daily latent Brownian process but only refreshes non-absorbing observed states on random refresh days, while still allowing death to occur on any day. It also supports patient-specific random drift heterogeneity through `mu_drift_sd`, which can generate more persistent differences in individual trajectory slopes while retaining the reduced-switching gap mechanism.
-   **Deterministic Simulator** (`sim_trajectories_deterministic`): Generates trajectories based on deterministic curves (linear + quadratic) with added noise, useful for specific stress tests.

### 2. Statistical Models & Robust Inference

This component extends functionality from `VGAM` and `rms` to ensure valid statistical inference in longitudinal settings where observations are correlated.

-   **Robust Covariance** (`R/robcov_vglm.R`): Implements the Huber-White "sandwich" estimator for `vglm` objects. This is critical because standard errors from standard software assume independence, which is violated in longitudinal data. It supports cluster-robust standard errors to account for within-patient correlation.
-   **Unified Interface** (`R/power.R`, `R/misc.R`): The `tidy_po` function provides a standardized way to extract coefficients, standard errors (robust or naive), and p-values from various model objects (`vglm`, `orm`, `clm`, `coxph`, `glm`), abstracting away package-specific differences.

### 3. State Occupancy Probabilities (SOPs)

The `soprob_markov` function (`R/sops.R`) is the workhorse for estimating probability mass distribution over states over time.

-   **Logic**: It iterates a transition matrix forward in time using the Law of Total Probability.
    -   $P(S_t = k) = \sum_j P(S_{t-1} = j) \times P(S_t = k | S_{t-1} = j)$
-   **Vectorization**: The implementation is highly vectorized. It constructs an "expanded" dataset containing all possible transitions for all patients at a given time step, allowing a single `predict()` call to generate the entire transition matrix.
-   **Fast Path**: For `vglm` models, `markov_msm_build` and `markov_msm_run` pre-compute design matrices by time point and previous state to bypass the slow `predict()` method during simulation-based inference. Inline transforms such as `rms::rcs(time, 4)` reuse the fitted model terms, while explicit precomputed basis columns can still be supplied via `t_covs`.

### 4. Marginalization & G-Computation

The `avg_sops` function (`R/sops.R`) implements G-computation to estimate population-average (marginal) effects.

1.  **Counterfactuals**: It creates copies of the baseline data for each treatment group (e.g., all patients treated vs. all control).
2.  **Projection**: It projects these cohorts forward using the fitted Markov model.
3.  **Averaging**: It averages the individual SOPs to obtain the marginal probability of being in state $k$ at time $t$.

## Inference Workflows

The `inferences()` function serves as a unified entry point for estimating uncertainty (`R/sops.R`).

### Method A: Simulation Engines (No Refit)

-   **MVN Engine** (`engine = "mvn"`): Draws coefficient vectors from a Multivariate Normal distribution $\beta^* \sim N(\hat{\beta}, \Sigma_{robust})`.
-   **Score-Bootstrap Engine** (`engine = "score_bootstrap"`): Uses cluster-level multiplier perturbations of score contributions with a one-step Newton update for coefficients. In v1, this is implemented for `robcov_vglm` objects in `avg_sops() |> inferences()` pipelines.
-   **Efficiency**: Both engines reuse the "Fast Path" (see above) to update predictions without refitting the model or reconstructing design matrices.
-   **Use Case**: Primary method for interactive analysis; score-bootstrap captures uncertainty from both coefficients and empirical patient mix in marginal (`avg_sops`) inference.

### Method B: Non-parametric Bootstrap

-   **Mechanism**: Resamples patients with replacement, refits the model from scratch, and re-calculates G-computation estimates.
-   **Robustness**: Does not rely on asymptotic normality assumptions for the coefficients but is computationally expensive.
-   **Use Case**: Validation of MVN method, small sample sizes, or highly non-linear derived quantities.

## Operating Characteristics Framework

Located in `R/power.R`, this system facilitates large-scale simulation studies to assess trial power, Type I error, and bias.

-   **Arrow Integration**: Uses `sample_from_arrow` to efficiently draw samples from massive super-population datasets stored on disk (Parquet/Arrow), enabling realistic "virtual trial" resampling.
-   **Orchestration**: `assess_operating_characteristics` manages the simulation loop, applying multiple analysis methods to the same dataset iteration to ensure fair comparisons.
-   **Separation of Concerns**: Fits and summaries are decoupled; summary statistics are aggregated via `summarize_oc_results`.
